En tant qu'expert en assurance qualité et ingénieur en automatisation QA, voici les cas de test "Boîte Noire" basés sur la User Story fournie, en respectant le format demandé.

| ID | Description | Pré-conditions | Étapes de test | Résultat Attendu |
|----|-------------|----------------|----------------|------------------|
| TC-CONN-001 | Connexion réussie avec des identifiants valides. (Happy Path) | Un utilisateur enregistré existe avec un email et un mot de passe valides. L'utilisateur est sur la page de connexion. | 1. Naviguer vers la page de connexion.<br>2. Entrer un email valide dans le champ "Email".<br>3. Entrer le mot de passe valide correspondant dans le champ "Mot de passe".<br>4. Cliquer sur le bouton "Se connecter". | L'utilisateur est redirigé vers son tableau de bord. |
| TC-CONN-002 | Tentative de connexion avec un email invalide/non-existant. (Negative Testing) | L'utilisateur est sur la page de connexion. | 1. Naviguer vers la page de connexion.<br>2. Entrer un email qui n'existe pas dans le système (ex: `utilisateur.inconnu@exemple.com`) dans le champ "Email".<br>3. Entrer un mot de passe quelconque (ex: `password123`) dans le champ "Mot de passe".<br>4. Cliquer sur le bouton "Se connecter". | Un message d'erreur "Utilisateur inconnu" s'affiche sur la page. |
| TC-CONN-003 | Tentative de connexion avec un champ mot de passe vide. (Edge Case) | L'utilisateur est sur la page de connexion. Le champ "Mot de passe" est vide par défaut. | 1. Naviguer vers la page de connexion.<br>2. Entrer un email valide (ex: `user.valid@exemple.com`) dans le champ "Email".<br>3. Laisser le champ "Mot de passe" vide.<br>4. Vérifier l'état du bouton "Se connecter". | Le bouton "Se connecter" est désactivé et ne peut pas être cliqué. |
| TC-CONN-004 | Tentative de connexion avec un mot de passe invalide pour un email valide. (Negative Testing Supplémentaire) | Un utilisateur enregistré existe avec un email valide. L'utilisateur est sur la page de connexion. | 1. Naviguer vers la page de connexion.<br>2. Entrer un email valide (ex: `user.valid@exemple.com`) dans le champ "Email".<br>3. Entrer un mot de passe incorrect (ex: `mauvaisMDP123`) dans le champ "Mot de passe".<br>4. Cliquer sur le bouton "Se connecter". | Un message d'erreur approprié s'affiche (ex: "Email ou mot de passe incorrect", "Mot de passe invalide"). |
